다음 요구사항을 만족하는 웹사이트를 구현하세요.

모든 문서는 한국어로 작성하며, 설계서를 읽고 모호한 부분은 구현 전에 질문하세요.

## 프로젝트 개요

그림 커미션(비상업적 그림 작업 의뢰)을 맡길 때 필요한 자료를 쉽게 만들 수 있는 웹사이트를 제작합니다.

사용자가 자료에 사용할

- 캐릭터 이미지
- 설명 텍스트
- 컬러 팔레트

등을 입력하면 **아티스트가 의뢰를 이해할 수 있도록 정리된 "한 장의 자료 시트"**를 생성할 수 있어야 합니다.

사용자는 이 자료 시트를 **PNG 이미지로 다운로드**할 수 있어야 합니다.

이 사이트는 **로컬 브라우저에서만 동작하며 서버나 DB를 사용하지 않습니다.** (vercel 배포 사용 예정)

---

# 기술 스택

다음 조건을 반드시 지켜주세요.

- React
- Vite
- TailwindCSS

추가 규칙

- 싱글 페이지 앱 (라우팅 없음)
- 컴포넌트 분리 가능
- 재사용 가능한 로직은 custom hook으로 분리
- 주석은 필요 시 한국어로 작성
- 서버 / DB 사용 금지
- 사용자가 업로드한 파일은 서버로 전송되지 않음

---

# 에셋 정의

이 사이트에서 **에셋(asset)** 이란 다음을 의미합니다.

- 이미지
- 텍스트 박스
- 컬러 팔레트 블록

사용자는 이 에셋들을 캔버스 위에 배치하여 **자료 시트를 구성**합니다.

---

# 편집 캔버스

자료는 **편집 캔버스 위에서 제작됩니다.**

기본 규칙

- 캔버스 비율: **세로형 (1200 x 1600)**, **가로형 (1600x1200)** 중 택1
- PNG 저장 시 현재 캔버스 크기를 그대로 export
- 에셋은 캔버스 바깥으로 이동할 수 없음

---

# 에셋 기능

## 이미지 업로드

사용자는 다음 형식의 이미지를 업로드할 수 있습니다.

- PNG
- JPEG

이미지는 캔버스에 배치 가능한 에셋으로 추가됩니다.

업로드된 파일은

- 서버로 전송되지 않음
- Object URL 등으로 로컬 미리보기만 사용

---

## 텍스트 블록

텍스트 블록 타입

- 제목
- 소제목
- 본문
- 리스트
- 키워드 태그

사용자는 자유롭게 텍스트를 입력할 수 있습니다.

플레이스홀더 예시

- 성격
- 외관 설명
- 의상 설명
- 요청사항

---

## 컬러 팔레트

컬러 팔레트 블록은 다음 기능을 가집니다.

- 팔레트 개수: **1~5개**
- 색상 선택: **RGB 범위 color picker**
- 각 색상은 **원(circle) 형태**

원 크기

- S
- M
- L

캡션

- 사용자가 직접 입력
- 토글로 표시 여부 설정 가능

컬러 팔레트는 **하나의 블록 단위로 이동합니다.**

# 배치 모드

사용자는 두 가지 배치 방식을 사용할 수 있습니다.

## 1️⃣ 템플릿 모드

기본 제공 레이아웃에 맞춰 배치됩니다.

사용자가 첨부하지 않은 슬롯은 **빈칸으로 표시됩니다.**

### 1인 템플릿

구성

- 대표 이미지 1개
- 전신/SD 이미지 1개
- 컬러 팔레트 3~5개
- 설명 섹션 3개

---

### 2인 템플릿

좌우 캐릭터 카드 구조

각 캐릭터 카드

- 대표 이미지
- 컬러 팔레트
- 설명 섹션

---

## 2️⃣ 자유 배치 모드

사용자는 에셋을 직접 캔버스에 배치할 수 있습니다.

배치 규칙

- 드래그로 이동 가능
- 자동 정렬 가이드 표시
- 같은 행/열에 스냅 정렬
- 에셋 겹침 금지
- 최소 마진 유지
- 캔버스 밖 이동 불가

드래그 구현은 다음 방식 사용 가능

- react-draggable
- react-rnd
- 또는 유사한 방식

# 스타일 커스터마이징

사용자는 다음 스타일을 변경할 수 있습니다.

- 전체 배경색
- 카드 배경색
- 테두리 색
- 제목 바 색
- 기본 폰트 (돋움 / 바탕)
- 기본 폰트 크기
- 둥근 모서리 정도
- 그림자 on/off

---

# 편집 흐름

사용 흐름

1️⃣ 에셋 업로드 및 생성
2️⃣ 배치 모드 선택 (템플릿 / 자유 배치)
3️⃣ 스타일 조정
4️⃣ 미리보기
5️⃣ PNG 다운로드

모드 전환은 **화면 이동 없이 메뉴에서 전환** 가능해야 합니다.

---

# PNG 다운로드

완성된 자료는 **PNG로 다운로드** 가능합니다.

구현 방식

- html-to-image
- dom-to-image

중 하나를 사용합니다.

# 코드 구조 요구사항

코드는 다음 기준을 지켜 작성하세요.

- 재사용 가능한 로직은 hook으로 분리
- UI 컴포넌트 분리
- 가독성 좋은 구조

예시 구조

```
components/
hooks/
canvas/
assets/
styles/
```
